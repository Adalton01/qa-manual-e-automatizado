# Cenários de Teste de Usabilidade e Interface

## CT-US001 - Validação da tela de Login

**Objetivo:**

Verificar se a tela de login apresenta uma experiência clara e fácil para o usuário.

**Passos:**

1. Acessar a tela de login.
2. Verificar os campos disponíveis.
3. Analisar organização dos elementos.
4. Realizar tentativa de login.

**Resultado esperado:**

- Campos de usuário e senha devem estar identificados.
- Botão de acesso deve estar visível.
- Mensagens apresentadas devem ser claras.

**Status:**

Passou

---

## CT-US002 - Validação de mensagens de erro

**Objetivo:**

Verificar se o sistema apresenta mensagens compreensíveis quando ocorre uma falha.

**Cenário:**

Usuário informa senha incorreta.

**Resultado esperado:**

Sistema deve informar que os dados estão incorretos sem apresentar mensagens técnicas confusas.

**Status:**

Passou

---

## CT-US003 - Responsividade da interface

**Objetivo:**

Verificar se a aplicação mantém boa utilização em diferentes tamanhos de tela.

**Cenário:**

Acessar o sistema em diferentes resoluções.

**Resultado esperado:**

Elementos da interface devem permanecer organizados e funcionais.

**Status:**

Não validado

# Conclusão

Os testes de usabilidade, interface e acessibilidade são importantes dentro do processo de QA, pois garantem que o software não apenas funcione corretamente, mas também proporcione uma experiência eficiente, clara e acessível para diferentes usuários.
