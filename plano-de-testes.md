# Plano de Testes - Sistema de Login

## 1. Introdução

Este plano de testes apresenta a estratégia, o escopo, os recursos e a abordagem utilizada para validar a funcionalidade da tela de login em um projeto de estudo de Qualidade de Software (QA).

O objetivo é documentar o processo de testes e demonstrar a aplicação de boas práticas de qualidade.

## 2. Objetivos do Teste

Garantir que a funcionalidade de login opere conforme o comportamento esperado, permitindo acesso aos usuários com credenciais válidas e impedindo tentativas de autenticação inválidas.

## 3. Escopo

Serão realizados testes para validar:

- Campos de usuário e senha.
- Login com credenciais válidas.
- Login com credenciais inválidas.
- Mensagens de erro apresentadas ao usuário.
- Validação de campos obrigatórios.
- Comportamento do botão "Entrar".
- Recuperação de senha (quando aplicável).

## 4. Itens a serem testados

- Interface da tela de login.
- Campos obrigatórios.
- Validação das informações inseridas.
- Processo de autenticação.
- Mensagens de retorno ao usuário.

## 5. Itens fora do escopo

Não serão testados neste projeto:

- Funcionalidades disponíveis após o login (dashboard e demais telas).
- Integrações com sistemas externos.
- Regras internas do servidor.

## 6. Critérios de Entrada

Para iniciar os testes, é necessário:

- Sistema disponível para execução dos testes.
- Ambiente de teste configurado.
- Dados de teste preparados.
- Cenários de teste definidos.

## 7. Critérios de Saída

Os testes serão considerados concluídos quando:

- Todos os casos de teste planejados forem executados.
- Resultados dos testes forem documentados.
- Possíveis problemas encontrados forem registrados.
- Não existirem defeitos críticos impedindo o funcionamento da aplicação.

## 8. Estratégia de Teste

Serão aplicadas as seguintes abordagens:

- Testes funcionais.
- Testes exploratórios.
- Validação de cenários positivos e negativos.
- Verificação das mensagens exibidas pelo sistema.

- Planejamento dos testes.
- Criação dos casos de teste.
- Execução dos cenários definidos.
- Registro dos resultados.

## 11. Riscos

Possíveis riscos identificados:

- Alterações no sistema durante a execução dos testes.
- Indisponibilidade do ambiente de teste.
- Falta de dados adequados para validação.
